子网

默认子网

root@ip-10-233-0-38:~# kk get subnets
NAME          PROTOCOL   CIDR            PRIVATE   NAT     DEFAULT   GATEWAYTYPE   USED   AVAILABLE
join          IPv4       100.64.0.0/16   false     false   false     distributed   1      65534
ovn-default   IPv4       10.222.0.0/16   false     true    true      distributed   4      65531

Kube-OVN中以Subnet来组织IP，每个NS可以归属到不同的子网里

NS下的Pod会自动从所属子网中获取IP，并共享网络配置（CIDR，网关类型，访问控制，NAT控制）

Kube-OVN中会预设一个默认子网，未显示声明子网配置的NS会自动从默认子网中划分IP，并使用默认子网的网络配置。

其中，默认子网使用分布式网关并对出口流量进行NAT转换，其行为与flannel的默认行为一致。

默认子网为，spec中deault为true的子网，一个集群下默认子网只能有一个，且创建后不可修改

Node子网

Kubernetes规定，所有Node都必须和Pod可以直接通信。

为了达到这个目的，Kube-OVN床架了一个join子网，并在每个Node节点创建了一块虚拟网卡ovn0接入join子网，通过该网络完成主机和Pod的通信。

自定义子网

一个或多个NS可以被绑定到一个子网中。

创建子网

root@ip-10-233-0-38:~# cat <<EOF | kubectl create -f -
> apiVersion: kubeovn.io/v1
> kind: Subnet
> metadata:
>   name: sn1
> spec:
>   protocol: IPv4
>   cidrBlock: 10.100.0.0/16
>   excludeIps:
>   - 10.100.0.1..10.100.0.10
>   gateway: 10.100.0.1
>   namespaces:
>   - ns1
> EOF
subnet.kubeovn.io/sn1 created

root@ip-10-233-0-38:~# kubectl create ns ns1
namespace/ns1 created
root@ip-10-233-0-38:~# kubectl run nginx --image=nginx:alpine -n ns1
